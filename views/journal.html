The State of AngularJS Routing

AngularJS is structured in a modular fashion for ease of both maintenance & dependency management. Moreover, the design is modular to facilitate an opportunity for developers to create 3rd party modules that extend, enhance or replace core Angular modules. Examples of this can be found on ngModules.org; popular community modules including Restangular and UI-Router. Both of which are designed to replace core angular modules while hopefully enhancing your developer experience in the process. This seems to be the primary goal of 3rd party modules, as none are aimed at replacing core Angular functionality out of spite or disagreement in design. When analyzing the 3rd party modules available for extending/enhancing/replacing Angular's core $route engine I find some divergence in the community offered solutions.

The problem with $route

The core URL routing engine, $route, does not support nested views or persistent state-based management of them. Developers seeking this type of hierarchical view structure have no choice but to utilize community modules that augment the core $route engine and the $routeProvider.
